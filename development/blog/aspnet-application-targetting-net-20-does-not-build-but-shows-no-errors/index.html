ASP.Net Application targetting .net 2.0 does not build, but shows no errors

By Keith Jackson 05/11/2008

This problem has been perplexing me for about a week now and, despite much internet trawling, I could not find an immediate solution. The problem with the internet is it seems that people are far more willing to post up solutions than they are explanations which, in this instance, was frustrating. I eventually found an article which pointed me in the right direction, but not without pain.

The scenario I was working with was a series of libraries referenced by an ASP.Net application. The libraries were a mixture of C# and VB.Net and the ASP.Net application was written in VB.Net.

I kept finding that, if I referenced an extension method, the build would fail; but only when used within the ASP.Net project code itself, the libraries themselves would compile fine.

The article that I eventually tracked down suggested that this is due to the dependency that ASP.Net has on the VB.Net runtime compiler, which causes the .net 2.0 hack for extension methods to fail within the scope of the web project itself.

If anyone has any other solutions to this then please post them up, they would be most welcome. However, what I did was ensure that my extension methods were in obvious static class constructs so that rather than calling myString.DoThis() in the web application I would just use the longhand way of calling StringEx.DoThis(myString) rather than using the method as an extension.

Problem solved, albeit in an annoying way. The article indicated that the problem may be specific to the VB.Net compiler, so whether the same problem would be found in a C# Web Site I cannot be certain, but if anyone has any info on this please post it up.
