ReflectionHelper – Reflection for the rest of us

By Keith Jackson 08/02/2011

ReflectionHelper – Reflection for the rest of us

By Keith Jackson 08/02/2011

As I said in an earlier post, I always found the .net Reflection methods quite unwieldy and unintuitive. To that end I wrote a suite of classes to make use of Reflection much easier. These are all wrapped up within the ReflectionHelper class within the MOT Framework Extensions [retired - see the link below to the isolated component]. In order to get access to the classes add a reference in your code to the Ministry base class library and add using Ministry.Reflection to your using statements. If you're using .net 4.0 then the DLR may be a more viable option than traditional reflection depending on your needs.

Fields

The following methods exist for fields...

  • ReflectionHelper.Fields.FieldExists() - Traditional and generic variants of this method can be called either passing in a type or a value of a given type along with a field name to look for on that type.
  • ReflectionHelper.Fields.GetField() - Traditional and generic variants of this method can be called by passing in an object to query along with a field name. For example; ReflectionHelper.Fields.GetField(myObj, "name");. Overloads are also provided for static types that take a type parameter instead of an object instance.
  • ReflectionHelper.Fields.SetField() - Traditional and generic variants of this method can be called by passing in an object to query along with a field name to set. For example; ReflectionHelper.Fields.SetField(myObj, "name", "new value");. Overloads are also provided for static types that take a type parameter instead of an object instance.
  • ReflectionHelper.Fields.GetFieldInfo() - Sometimes you do need the Info objects but in traditional .net code this can become quite awkward if the object you are looking for is further down the inheritance tree than the layer that you are interrogating. The ReflectionHelper wraps up all of the code required to recurse down the inheritance tree to find the type that you are looking for. This method also provides the same service to the rest of the Fields class, enabling the helper to provide Reflection against the whole inheritance tree.

Properties

Reflection on Fields & Properties are similar. The following methods exist for properties...

  • ReflectionHelper.Properties.PropertyExists() - Traditional and generic variants of this method can be called either passing in a type or a value of a given type along with a property name to look for on that type.
  • ReflectionHelper.Properties.PropertyIsReadOnly() - Traditional and generic variants of this method can be called either passing in a type or a value of a given type along with a property name to look for on that type.
  • ReflectionHelper.Properties.GetProperty() - Traditional and generic variants of this method can be called by passing in an object to query along with a property name. For example; ReflectionHelper.Properties.GetProperty(myObj, "Name");. Overloads are also provided for static types that take a type parameter instead of an object instance.
  • ReflectionHelper.Properties.SetProperty() - Traditional and generic variants of this method can be called by passing in an object to query along with a property name to set. For example; ReflectionHelper.Properties.SetProperty(myObj, "Name", "new value");. Overloads are also provided for static types that take a type parameter instead of an object instance.
  • ReflectionHelper.Properties.GetIndexerItem() - Traditional and generic variants of this method can be called by passing in an object to query along with a property name and an index. For example; ReflectionHelper.Properties.GetIndexerItem(myObj, "Index", 4);.
  • ReflectionHelper.Properties.SetIndexerItem() - Traditional and generic variants of this method can be called by passing in an object to query along with a property name and an index to set. For example; ReflectionHelper.Properties.SetIndexerItem(myObj, "Index", 4, "new value");.
  • ReflectionHelper.Properties.GetPropertyInfo() - Sometimes you do need the Info objects but in traditional .net code this can become quite awkward if the object you are looking for is further down the inheritance tree than the layer that you are interrogating. The ReflectionHelper wraps up all of the code required to recurse down the inheritance tree to find the type that you are looking for. This method also provides the same service to the rest of the Properties class, enabling the helper to provide Reflection against the whole inheritance tree.

Methods

The following methods exist for methods. Method access through Reflection is very problematic. The Methods class within the ReflectionHelper, as with Fields and Properties, benefits from inheritance tree recursion.

  • ReflectionHelper.Methods.ExecuteMethod() - Many overloaded variants of this exist to call a method with or without parameters. The method execution code will search for the method to execute using the method below.
  • ReflectionHelper.Methods.GetMethodInfo() - Sometimes you do need the Info objects but in traditional .net code this can become quite awkward if the object you are looking for is further down the inheritance tree than the layer that you are interrogating. The ReflectionHelper wraps up all of the code required to recurse down the inheritance tree to find the type that you are looking for.
  • ReflectionHelper.Methods.GetMethodParameters() - Provides you with the info about the parameters for a given method.

That's the key functionality of the ReflectionHelper classes. If you want to do any kind of deep reflection and you don't have the joy of the DLR to help you out then I hope you'll find some of the methods in these classes really useful. If you want to find out any more, simply check out the source code.
