UI Automation for .net

By Keith Jackson 07/05/2011

I've spent a good few years of my professional career working on a variety of test frameworks merging many different techniques to do UI Automation testing of various types. As a primarily .net focussed coder I always felt that this was an area that was severely lacking - until now!

Along came WebDriver

In the bad old days there was Watin and life sucked until you learnt Ruby (then you had WatiR, which was actually pretty good) if you wanted to stick with .net then your only alternative to Watin was Selenium, which was pretty limited for all but basic automation, but then came Selenium 2, currently still in beta, which utilises WebDriver for automating a variety of browsers using a common interface. The exposed .net API is fry simple to use and is split across a variety of interfaces with really clear separation of concerns. Constructing a site specific test framework on top of this is fairly painless and takes a minimal amount of time. Tests are very easy to write and clear, they lack some of the verbosity offered by Ruby, but that's more of a language feature than anything else. The software is still in beta but already supports IE, FireFox and Chrome. Whether other browser support will follow remains to be seen, but adding it shouldn't be too hard due to the design of the API.

The current version is lacking some fairly obviously useful methods and I'll be publishing up some code to add to what's currently on offer at some point over the next few weeks, but I'm sure that many of these gaps will be filled as the software continues to move further out of beta.
